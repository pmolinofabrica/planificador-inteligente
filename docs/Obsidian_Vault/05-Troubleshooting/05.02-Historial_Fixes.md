# Histórico Reciente de Problemas Resueltos

Bitácora de Fixes masivos documentados para referencia:

## Fix 1: Optimización de Consultas API (Fetch Limit)
Se descubrió que vistas críticas omitían data porque la API PostgREST por default aplica límite de 1000 items al consultar directo sin filtros estrictos `.eq()` sobre campos referenciados (`convocatoria`). Se implementó un barrido filtrado de `id_plani` acotado al Set real.

## Fix 2: Limpieza de Matrices Fantasmas en Frontend
La vista de `Aperturas` sobre-contaba cupos y llenaba el log visual sumando los totales por múltiples iteraciones debido a un contador global impuro y un fallback al turno `1` (cuando Apertura era `45`). Se saneó con variables locales temporales puras de TS.

## Fix 3: Persistencia Segura de "Tipo Organización"
**Bug Crítico:** Configurar "Rotación Completa" fallaba silenciosamente en días vacíos (sin residentes), porque el atributo sobrevivía incrustado en la tabla hija transaccional `menu_semana`. Si el supervisor vaciaba el turno, se formateaba la configuración general.
**Solución Técnica:** Se fragmentó normalizando a una tabla `configuracion_turnos` para aislar el nivel macro "Día + Turno" de la data de la persona individual. La capa de presentación `DevicesTab` usa un `.upsert()` universal con herencia dinámica según el filtro actual visible (`3`, `4`, `45`).
