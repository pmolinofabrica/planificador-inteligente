# Errores y Bugs Comunes

Un registro de los "Gotchas" que retrasaron el desarrollo sistemáticamente en este stack tecnológico.

## 👻 1. "El Rechazo Invisible" (Error 42501 Permission Denied)
Supabase (PostgREST) enmascara los errores de permisos fallando en silencio o tirando genéricos 400 Bad Request si la operación llega por frontend.
**Solución:**
Toda tabla creada por consola `psql` REQUIERE permisos explícitos:
```sql
GRANT SELECT, INSERT, UPDATE, DELETE ON public.mi_tabla TO authenticated, anon;
```
Luego **DEBEMOS reiniciar el cache REST**: `NOTIFY pgrst, 'reload schema'` si no lo toma enseguida.

## ⏮️ 2. "El Click de Rebote" (Race Condition Optimista)
La PWA usa mutaciones optimistas. Si das click, la UI cambia al instante y se envía el commit a la BBDD. Si falla silenciosamente (por ej. por un espacio de sintaxis en `onConflict` de Supabase o un error de permisos como el #1), el código tira `catch` y restaura el state al valor base anterior. 
A los ojos del usuario, es como si el click no surtiera efecto y la celda "rebotara".  **Siempre asume un throw oculto de DB en este escenario.**

## 🔢 3. Incompatibilidad Date-Fns vs UI (El Bug de los Ceros)
`ActiveDates` se calcula cortando los ceros `[1/3, 2/3]` pero DB PostgreSQL despacha y traga puro pad ISO `[01/03]`. 
Perder de vista qué componente está usando el formato y guardarlo crudo rompe el "Map" y crea filas invisibles que no pintan datos pero llenan la DB.

## 🧱 4. El "Group" Heredado Destructor en Rotación
Al mover de la celda de Taller 1 de un día, a otra celda Taller 2 del mismo día en Rotación Completa, se detectó que el frontend mataba el `numero_grupo`. El query `.update()` debe incluir invariablemente la propagación del grupo original, o el agente pierde sus vínculos visuales A/B instantáneamente.
