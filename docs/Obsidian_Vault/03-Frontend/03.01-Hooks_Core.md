# Hooks Core: `useAssignmentData.ts`

El cerebro absoluto del frontend es el hook `useAssignmentData.ts`. Toda la lógica de extracción, transformación y cruce de datos masivo ocurre aquí adentro para alimentar a los componentes de UI.

## 🧠 Filosofía de Carga (Data Fetching)
Dado que la aplicación muestra una matriz mensual de asignaciones, el frontend no hace peticiones individuales por cada celda. El hook realiza un **gran bloque de llamadas en paralelo** (`Promise.all`) a Supabase al montar el mes, y luego cruza esa información en memoria usando diccionarios veloces (`Maps`/`Records`).

### Proceso de Carga:
1. **Resolución de Fechas:** Usa `getMonthParts()` para derivar rangos ISO (ej. `2026-03-01` a `2026-03-31`).
2. **Setup Base:** Descarga los catálogos estáticos: `dispositivos`, residentes activos (`datos_personales`) filtrados por la cohorte actual.
3. **Mapeo de Capacitaciones:** Baja todo el cruce histórico de capacitaciones del año y se lo entrega al [[03.04-Utilidades|caps-builder]] para procesarlo.
4. **Matrices Centrales:** Carga `menu`, `menu_semana`, `configuracion_turnos`, y `calendario_dispositivos`.

## 📦 Estructuras de Estado Entregadas (`AssignmentsMatrix`)

Una vez la base de datos devuelve las filas planas, el hook transforma la data agrupándola por `fecha` y luego por `id_dispositivo`.

El formato clave devuelto (`assignmentsDb`) es de Tipo jerárquico:
```typescript
{
  "01/03": { // Fecha (formateada en string corto sin 0 a la izquierda)
    "5": [   // ID de Dispositivo
      { id: 124, name: "Juan Perez", score: 1000, numero_grupo: 1, acompana_grupo: false }
    ]
  }
}
```

### Mapas Auxiliares Críticos:
- `calendarDb`: Guarda los *cupos meta* de cada dispositivo por fecha. (Ej. `calendarDb["01/03"]["5"] = 3` cupos).
- `convocadosDb`: Quien debe asistir ese día.
- `inasistenciasDb`: Veto total. Si un residente está aquí, el frontend lo tacha y contabiliza como falta.
- `tipoOrganizacionMap`: Estado vital (`fija`, `rotacion simple` o `completa`) dictado desde `configuracion_turnos` por el cual los paneles saben qué lógica de "Quitar" usar.

## ⚠️ Advertencias y Mejores Prácticas Históricas
1. **El Bug de los Ceros:** Supabase PostgreSQL devuelve fechas ISO con ceros `01-03`, pero este frontend usa internamente y compara strings formateados sin ceros `1/3` (función `formatUiDate`). Si cruzas data nueva, NUNCA olvides pasar tus fechas de DB por `formatUiDate()` antes de guardarlas en el estado, o se crearán matrices invisibles (Data Drift).
2. **Paginación Convocatoria:** Cuando el hook fetchea datos gigantescos como historial, NUNCA se hace `select('*')` a menos que pases una variable de límite por fecha explícitamente usando la tabla `planificacion` como pivote (ver bug fix en Historial).
