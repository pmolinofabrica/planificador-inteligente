# Utilidades y Librerías

El peso algorítmico independiente se aísla en archivos separados bajo `src/utils` y `src/lib`.

## 🛠 `caps-builder.ts`
El motor que parsea las asistencias históricas y cruza tabuladores.
**¿Por qué existe?** PostgREST no es muy rápido en devolver queries hiper anidadas (un residente -> una cap X -> 5 asistencias en meses Y -> válido o inválido).
Esta utilidad se procesa en O(N) recorriendo las 3 tablas interconectadas del backend (`capacitaciones`, `_participantes`, `_dispositivos`) para resolver rápidamente un Map booleano en frontend: 
```typescript
capsMap[`${idAgente}-${idDispositivo}`] = true | false
```
Si el map retorna falso para el residente `45` en el local `7`, `PlanningMatrix` renderizará un logo de restricción `⚠️`.

## 📆 `dateUtils.ts`
Suministrado para resolver operaciones mundanas como generar el ciclo lectivo escolar (Marzo -> Diciembre), convertir meses al español, y lidiar con los desfasajes timezone.
Importante uso para estandarizar `01-03` contra `"01/03"`.

## 🎨 `floor-utils.ts`
Estilización pre-determinada (Tailwind) orientada al "Molino".
Retorna cadenas de clases con los prefijos correctos (Papel, Madera, Textil) para el sombreado dinámico, para no contaminar los archivos UI puros con condicionales de color kilométricos.
