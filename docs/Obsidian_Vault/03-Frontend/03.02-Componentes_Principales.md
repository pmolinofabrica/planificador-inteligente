# Componentes Principales

La interfaz gráfica pivota alrededor del state central entregado por el hook. Aquí están los componentes neurálgicos que debes conocer si necesitas alterar la UI.

## 🗺️ Vistas de Grilla (Los "Tabs" grandes)

### 1. `PlanningMatrix.tsx` (Matriz Mensual)
El componente más visual. Renderiza una matriz estilo Excel cruzando `activeDates` (eje X) con `dbDevices` agrupados por pisos (eje Y).
- **Especialidad:** Cruza asignaciones vs parámetros de capacidad para pintar advertencias visuales (Rojo intenso si falta gente, Verde si el cupo está lleno).
- **Indicadores:** Aquí es donde se pintan los iconos `⚠️` de Sin Capacitación de residentes (calculado vía el `capsMap`).

### 2. `ExecutionTab.tsx` (Tablero Kanban de Apertura)
Vista moderna de tarjetas diseñada para el "Día a Día". Es una abstracción *por fecha única*.
- **Especialidad:** Divide los componentes en "Libres", "Dispositivos X, Y, Z" y "Descanso/Ausentes".
- Muestra el esquema Drag & Drop clásico (aunque lógico, de momento mediante clics en páneles laterales). 
- Importa sus variables de color (Grupo A y B) directamente desde las variables `--group-a-text` de CSS utilizando dependencias inyectadas en línea.

### 3. `MenuView.tsx` (Modo Kiosco / Lectura)
Versión limpia, "read-only" de Planning/Execution con el bloqueo anti-ediciones. La capa de presentación es final. Incluye soporte paramétrico especial para contingentes visitantes (`VisitBlock`).

### 4. `DevicesTab.tsx` (Editor de Cupos)
Donde el usuario define cuánta gente se necesita por equipo.
- Cruza `.update` y `.upsert` contra las tablas `calendario_dispositivos` y `configuracion_turnos`. Las operaciones aquí no asignan personas, configuran la matriz estructural de un día.

## 🏗️ Páneles de Contexto (Sidebars)

Gran parte de la usabilidad reside en componentes "Flotantes" (Renderizados a la derecha o izquierda en el layout de `Index.tsx`) que reaccionan a lo que el usuario seleccionó.

### `ResidentSidebar.tsx`
Se dispara cuando el usuario toca el nombre de un **residente**.
Permite "Cambiar", "Quitar" o "Mover" al agente.
- **Dato clave:** Sus lógicas de base de datos son complejas. Para quitar o cambiar en modo *Rotación*, ejecuta queries exactas contra la Key de Supabase (`id_menu_semana`) porque la rotación puede significar que la misma persona esté en el Taller 1 de 10hs a 13hs y en el Taller 2 de 13hs a 16hs en el *mismo* turno.

### `CellSidebar.tsx`
Se dispara cuando el usuario hace clic en el **título de un dispositivo** en una fecha o en un **hueco vacío**.
Muestra quiénes "ya están" y quiénes están "libres".
- Realiza el `.insert()` central a la DB para asignar.
- Si hay modales de alerta (Ej "Se superó el límite", o "¿Añadir cupo?"), nacen aquí.

### `VacantsSidebar.tsx` / `VacantActionSidebar.tsx`
Sub-paneles específicos para atajos de "llenado rápido" cuando quedan huecos libres (Libres).
