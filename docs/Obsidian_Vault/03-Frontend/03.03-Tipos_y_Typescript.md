# Tipos y TypeScript

El proyecto hace fuerte uso de los estandartes tipificados de ES6 / TS. 

## 🔌 Generación Inyectada (Supabase Types)
El archivo **`src/integrations/supabase/types.ts`** es el "Source of Truth" de la base de datos real. 
- Funciona exportando interfaces planas (interfaces Json, Interfaces DB).
- Todo cambio de `CREATE TABLE` en Supabase **debe ser correspondido** con un update manual en este archivo (Ej: Cuando añadimos `configuracion_turnos`, tuvimos que agregar la llave para evitar que el CI de TypeScript llorara).
- Si usas la CLI oficial: `npx supabase gen types typescript --project-id XYZ > types.ts`.

## 📦 Tipos de Dominio Frontend
El archivo **`src/types/assignments.ts`** maneja la abstracción del negocio (cómo presenta localmente React a un dispositivo o residente).

Componentes vitales:
```typescript
export interface AssignmentEntry {
  id: number;
  name: string;
  score: number;        
  numero_grupo: number | null; 
  acompana_grupo: boolean;
}

export type AssignmentsMatrix = Record<string, Record<string, AssignmentEntry[]>>;
// Retorna => [fecha][dispositivoID] => Array[]
```

**Nota para debugging:**
Cuando recibas un error del linter que grite: *Parameter X is missing from '{...}'*, probablemente es porque creaste un alias o te olvidaste de agregar la columna base al Request `.select()` en Supabase. TypeScript aquí deduce el tipo de forma muy fina partiendo de las Promesas.
