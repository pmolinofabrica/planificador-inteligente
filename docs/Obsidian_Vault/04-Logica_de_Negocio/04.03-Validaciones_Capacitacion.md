# Validaciones de Capacitación

La métrica principal de control de calidad del Molino es asegurarse que los Agentes estén capacitados para operar la maquinaria o dictar el taller asignado. 

## ⚙️ El Motor: `capsMap`
El `useAssignmentData` entrega un diccionario llamado `agentGroups` y un proxy auxiliar que la gente denomina internamente como `capsMap`. 
En la refactorización reciente, el constructor se aisló puramente llamando la exportación `buildResidentCaps()` (en `lib/caps-builder.ts`).

### El camino de los datos:
1. `capacitaciones_participantes` trae el booleano `asistio = true/false` de todos los residentes en todas las charlas.
2. El builder lo cruza y retorna un Record: `"AgenteID-DispositivoID" -> TRUE (Capacitado)`.

## 🚧 Feedback en UI
Los componentes (Como `PlanningMatrix.tsx` y `ExecutionTab.tsx`) corren checks preventivos silenciosos.
- Buscan en su matriz la Key compuesta: `Const isTrained = !!capsMap[\`\${res.id}-\${device.id}\`]`.
- Si `isTrained === false`. Renderizan de inmediato un SVG de Advertencia junto al nombre del usuario en ese nodo (`⚠️`).
- **Bloqueos Duros vs Blandos:** Actualmente el UI aplica "Bloqueos Blandos". Reclama y marca en rojo o con simbolos la falta de capacitación, pero *no lanza throws técnicos ni bloquea al usuario supervisor* para que asigne forzadamente de ser necesario en contingencias reales.
