# Visión General y Arquitectura

## 💻 Stack Tecnológico

El proyecto utiliza un stack moderno y reactivo:

- **Frontend Core:** React 18 + TypeScript + Vite.
- **Routing:** React Router v6.
- **Estilos:** Tailwind CSS + Variables de CSS Nativas (Para temáticas complejas de colores).
- **Componentes Base:** Radix UI primitivas + shadcn/ui.
- **Iconografía:** Lucide React.
- **Backend as a Service (BaaS):** Supabase (PostgreSQL + PostgREST + Auth).
- **Gestión de Fechas:** `date-fns` y funciones propias en `src/utils/dateUtils.ts`.

## 🏗️ Filosofía de Arquitectura

1. **Estado Centralizado en Cliente (Fat Client):**
   La aplicación está fuertemente orientada a cargar toda la grilla de datos de un mes de una sola vez hacia la memoria del navegador.
   - El hook `useAssignmentData.ts` orquesta múltiples resoluciones en paralelo mediante `Promise.all` e hidrata diccionarios (Maps) en memoria (`assignmentsDb`, `calendarDb`, `tipoOrganizacionMap`).
   - Todos los componentes hijos (Tabs, Sidebars) reciben un hiper-objeto `data` que contiene estos estados y mutadores.

2. **Base de Datos Desnormalizada (Híbrida):**
   Para efectos de rendimiento y simplicidad de UI, hay tablas transaccionales (`menu`, `menu_semana`) que guardan llaves duplicadas para búsquedas de punto único.
   Existen también Vistas (`vista_*`) y RPCs (Funciones en la BD de Postgres) para reportes de cálculo pesado (Inasistencias, recuento de capacitados).

3. **Caché y Optimistic Updates (Actualizaciones Optimistas):**
   Las interacciones del usuario (asignar, eliminar, cambiar cupo) siguen un patrón optimista:
   - El frontend muta el estado de React *inmediatamente* asumiendo que el éxito es inminente.
   - Dispara el fetch a Supabase en modo silencioso (`.insert()`, `.update()`, `.upsert(), .delete()`).
   - Si Supabase lanza un error en el bloque `catch {}`, el frontend deshace el cambio local de React a su estado anterior.

4. **Variables de Diseño Semánticas:**
   Los colores de los grupos (A, B) y los pisos (Papel, Madera, Textil) no se manejan con colores crudos de Tailwind como `text-red-500`, sino con variables CSS dinámicas registradas en `index.css`: `[hsl(var(--floor-1-text))]`, `[hsl(var(--group-a-accent))]`.
